# Plan 04 — CTA and journey instrumentation (ANLY-05)

## Objective

Instrument **`window.TMAnalytics.track`** at **booking**, **ticket panel**, **gift cards**, **nav/group CTAs**, **locations**, and **missions** surfaces per **ANLY-05** and **`06-CONTEXT.md` JOURNEY** bullets — **two waves** of hook placement to ease review.

## Preconditions

- Plan 02: `TMAnalytics.track` implemented.
- Phase 5 booking behavior: same-tab Roller/booking where applicable.

## Tasks — Wave A (primary conversion)

### Task 4.1 — Ticket panel + Roller handoff

**Goal:** `track` on **panel open/close**, **location change** (slug only), **book click** (intent), **checkout_start** when navigating to `https` checkout URL.

**read_first:**

- `js/ticket-panel.js`
- `js/roller-checkout.js`
- `analytics-labels.json`

**Files:**

- `js/ticket-panel.js`
- optionally `js/roller-checkout.js`

### Task 4.2 — Nav / mobile menu CTA

**Goal:** `track` **nav_cta_click** with `cta_id` / `section` per labels.

**read_first:**

- `js/nav.js`

**Files:**

- `js/nav.js`

### Task 4.3 — Locations module

**Goal:** `track` **location_pick** when user selects a location (slug, region only — no address).

**read_first:**

- `js/locations.js`

**Files:**

- `js/locations.js`

## Tasks — Wave B (secondary surfaces)

### Task 4.4 — Gift card + group + missions links

use **delegated** listeners or existing CTA hooks; **track** **only** pre-navigation intents with **href path** or **label key**, not full URL if it contains PII (gift URLs should be **path only**).

**read_first:**

- Relevant templates or `src/components` for gift/groups/missions links

**Files:**

- As needed — prefer **one** delegated handler in `js/analytics.js` or `js/cta-delegation.js` to avoid duplicating listeners

### Task 4.5 — Dedup / rate-limit guard

**Goal:** Optional **100–300ms** dedupe for duplicate **click** bursts on same `event_id` pattern — document if implemented or intentionally omitted.

**Files:**

- `js/analytics.js`
