# Plan 03 — Netlify contact form: Astro + thank-you parity, form analytics

## Objective

Meet **FORM-01** through **FORM-04**: Astro **`/contact`** keeps **Netlify** form semantics (`data-netlify`, honeypot, `name` fields); **`/contact-thank-you`** route exists in **Astro output** matching static behavior; **non-PII** form lifecycle events via `TMAnalytics.track` (open/focus, validation error **count only**, submit attempt, success navigation).

## Preconditions

- `TMAnalytics.track` available (Plan 02).
- Static **`contact.html`** and **`contact-thank-you.html`** are reference for parity.

## Tasks

### Task 3.1 — Astro contact page audit + Netlify attributes

**Goal:** Ensure `src/pages/contact.astro` (or equivalent) includes required Netlify form attributes and `action`/`method` match static contract.

**read_first:**

- `contact.html`
- `src/pages/contact.astro`
- `_redirects` for thank-you

**Files:**

- `src/pages/contact.astro`

**Acceptance criteria:**

- [ ] `data-netlify="true"` (or current Netlify convention), honeypot field preserved.
- [ ] `action` posts to thank-you path consistent with static site.

**verify:**

```bash
npm run build:astro && rg -n "netlify|data-netlify" dist/contact.html 2>/dev/null || rg -n "netlify" dist/contact/index.html
```
