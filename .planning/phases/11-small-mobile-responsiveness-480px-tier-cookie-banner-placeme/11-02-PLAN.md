<objective>
Add a `@media (max-width: 480px)` tier to the 7 page-local CSS partials currently shipped in Astro that lack one (or have only a partial-scoped rule). Each tier stacks remaining multi-column grids to 1fr, reduces hero/section-title sizes that don't fit small phones, tightens horizontal padding so content doesn't overflow, and respects the tap-target floor enforced by `npm run check:tap-targets`.

Purpose: Close the page-content half of the launch-blocking responsive gap diagnosed in Phase 10 UAT test 8. Plan 11-01 handles shared chrome (nav/footer/faq/base/newsletter); this plan handles per-page hero, grid, and copy density. Together they bring the Astro build to small-mobile parity at 320–425px.

Output: 7 modified partial files, each containing a new `@media (max-width: 480px)` block (or a SECOND block in `birthdays-inline.raw.css.txt`).
</objective>
