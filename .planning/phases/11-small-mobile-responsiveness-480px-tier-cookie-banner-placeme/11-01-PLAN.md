<objective>
Add an explicit `@media (max-width: 480px)` tier to the 5 shared CSS files (`nav.css`, `footer.css`, `faq.css`, `base.css`, `newsletter.css`) so the chrome rendered on every page (header bar, footer grid, FAQ accordion, page-header, and embedded newsletter section) compresses correctly at 320–425px viewports without breaking visual parity or shrinking tap targets below the 44px / 48px floor enforced by `npm run check:tap-targets`.

Purpose: Close half of the launch-blocking responsive gap diagnosed in Phase 10 UAT test 8 (severity major). The shared CSS files currently jump from the 768px tier directly to 320px viewport, so chrome elements (notably the nav at the top and the footer columns at the bottom) overflow or fail to wrap on iPhone SE / small Android / foldables in single-pane mode.

Output: 5 modified CSS files, each containing a new `@media (max-width: 480px)` block appended at the end of the file. No new files. No changes to JS. No changes to tokens. `npm run check:tap-targets` continues to exit 0.
</objective>
