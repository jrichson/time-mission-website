<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external request -> `_redirects` | User- and crawler-supplied paths are redirected by static host rules. |
| route registry -> sitemap | Authored route data determines crawlable canonical URL entries. |
| docs -> operator behavior | Redirect docs influence future host and cutover decisions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | `_redirects` alias destinations | mitigate | Keep every destination internal and registry-listed; no external targets are introduced. |
| T-02-02 | Denial of Service | `_redirects` source/target graph | mitigate | Remove source-target loops (`/licensing`, `/orland-park`, `/dallas`, `/brussels`) and run `npm run check:routes -- --redirects`. |
| T-02-03 | Tampering | `sitemap.xml`, `scripts/check-sitemap.js` | mitigate | Generate expected sitemap locs from `src/data/routes.json` and reject `.html` or non-root trailing-slash locs. |
| T-02-04 | Tampering | redirect and sitemap edits | mitigate | Only change redirect rows, sitemap locs, and docs; do not edit CSS, IDs, classes, ARIA hooks, or visible page copy. |
</threat_model>
