| Property | Value |

## Route And Navigation Interaction Contract

| Surface | Contract |
|---------|----------|
| Canonical URLs | Every migrated public page uses a clean extensionless path with no trailing slash, except root `/`. |
| Legacy `.html` URLs | Every legacy `.html` route redirects directly to its matching clean route in one hop where the static host allows. |
| Internal links | First-party navigation links should use clean canonical paths, not `.html` URLs. |
| Navigation behavior | Existing nav, mobile menu, location picker, footer links, ticket panel triggers, FAQ interactions, and form flows must behave visually and interactively the same after href updates. |
| Query strings | Existing query-driven behavior such as `?book=1` must not be broken by clean route updates; implementation notes must document host query preservation behavior. |
| URL fragments | Destination fragments may remain where the target page supports them, but redirect source fragments must not be treated as matchable because browsers do not send them to the server. |
| Redirect status | Permanent legacy page moves should use explicit 301 status unless intentionally temporary and documented. |
| Analytics paths | Analytics page-path context should use the same clean no-trailing-slash contract as canonical URLs. |
| Sitemap and schema URLs | `sitemap.xml`, canonical tags, Open Graph URLs, and JSON-LD URLs must agree with the route registry. |

---

## No-Change Guardrails

- Do not change `css/variables.css`, shared CSS modules, page-local visual styles, font assets, image/video assets, or animation timing as part of Phase 02 unless a routing implementation absolutely requires a non-visual metadata update.
- Do not rename, remove, or restructure existing selectors, IDs, ARIA attributes, `data-*` attributes, script hooks, or DOM structures used by `js/nav.js`, `js/locations.js`, `js/ticket-panel.js`, `js/roller-checkout.js`, or `js/a11y.js`.
- Do not change ticket panel markup, overlay semantics, focus behavior, skip-link behavior, mobile menu controls, location persistence behavior, or booking CTA visible state.
- Do not introduce client-side JavaScript redirects to mask route contract gaps; source-authored URLs and static-host redirects must carry the contract.
- Do not redesign 404, contact thank-you, policy, location, group, or marketing pages during this phase.
