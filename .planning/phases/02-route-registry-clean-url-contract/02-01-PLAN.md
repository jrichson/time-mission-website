<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| registry -> redirect validation | Authored JSON controls redirect expectations and must not allow external or looping targets. |
| source files -> public URL surfaces | Authored HTML/JS/JSON/XML values become crawler, analytics, and user-visible navigation surfaces. |
| source -> dist | Astro build output must not drift from the source route contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | `src/data/routes.json`, `_redirects` aliases | mitigate | Reject external alias/redirect targets unless `allowExternal: true` and `externalAllowlistReason` are present; Phase 02 registry should contain only internal targets. |
| T-02-02 | Denial of Service | `_redirects` parser and redirect graph | mitigate | Detect source-target equality and source-to-target chains after stripping query/hash and fragments. |
| T-02-03 | Tampering | sitemap, canonical, schema, internal links | mitigate | Validate sitemap locs, canonical links, `og:url`, JSON-LD URL fields, and first-party hrefs against `src/data/routes.json`. |
| T-02-04 | Tampering | HTML/JS source sweep | mitigate | Validator only flags URL-bearing attributes/metadata; later tasks must preserve visible copy, CSS classes, IDs, ARIA hooks, script hooks, and DOM structure. |
</threat_model>
