<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source registry -> build sync | Authored route data determines which static HTML files enter `public/` and `dist/`. |
| build output -> static host | Generated `dist/` files and host files become deployable Cloudflare Pages output. |
| verifier -> release confidence | Passing checks determine whether Phase 02 can be treated as complete. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | registry-driven static copy | mitigate | Copy only internal `route.outputFile` values from `src/data/routes.json`; do not support external paths or absolute destinations. |
| T-02-02 | Denial of Service | built `_redirects` | mitigate | Run `npm run check:routes -- --dist` to detect loops/chains in built redirect output. |
| T-02-03 | Tampering | `dist/sitemap.xml`, `dist/*.html`, built canonicals | mitigate | Validate built output against the same registry after `npm run build:astro`. |
| T-02-04 | Tampering | static HTML copy into `public/` | mitigate | Copy files as route artifacts only; do not mutate visible HTML, CSS, classes, IDs, ARIA hooks, or script hooks during sync. |
</threat_model>
