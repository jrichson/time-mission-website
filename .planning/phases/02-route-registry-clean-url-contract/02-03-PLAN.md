<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source HTML -> browser navigation | Authored href values become user navigation paths. |
| source HTML -> crawlers/social parsers | Canonical, Open Graph, and JSON-LD URL fields are consumed by external systems. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | href and metadata URL changes | mitigate | Only use internal clean paths from `src/data/routes.json`; leave external URLs unchanged. |
| T-02-02 | Denial of Service | query/fragment link rewrites | mitigate | Preserve query strings and fragments on clean paths so no client-side routing loops are introduced. |
| T-02-03 | Tampering | canonical, `og:url`, JSON-LD URL fields | mitigate | Run `npm run check:routes -- --sources --scope=root` to compare URL surfaces to the registry. |
| T-02-04 | Tampering | top-level HTML source sweep | mitigate | Restrict edits to URL-bearing attributes/metadata and preserve visible copy, classes, IDs, ARIA hooks, script hooks, style blocks, and DOM structure. |
</threat_model>
