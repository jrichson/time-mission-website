## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Replaced import.meta.url with process.cwd() for repo root resolution**
- **Found during:** Task 2 build verification
- **Issue:** Astro's Vite build compiles all `.astro` pages AND lib files into bundled chunks in `dist/.prerender/chunks/`. At prerender time, `import.meta.url` inside both page frontmatter and lib modules resolves to the compiled chunk path (e.g. `file:///…/dist/.prerender/chunks/antwerp_BM6WuctJ.mjs`), not the source file. Git lookups on those paths return empty, causing every page to fall back to the build date.
- **Fix:** Changed `last-updated.ts` to use `process.cwd()` as the repo root anchor (stable — Astro always builds from the repo root). `canonicalPath` is mapped to `src/pages/{slug}.astro` via `canonicalToSourceFile()`. The layout already has `canonicalPath` as a required prop, so no new prop is needed on any page.
- **Files modified:** `src/lib/last-updated.ts` (rewrote repo-root strategy), `src/layouts/SiteLayout.astro` (pass `canonicalPath` to `resolveLastUpdated`)
- **Commit:** `68f25f1`

**2. [Rule 3 - Scope cleanup] Removed per-page sourcePageUrl={import.meta.url} additions**
- **Found during:** Task 2 refinement
- **Issue:** The initial implementation added `sourcePageUrl={import.meta.url}` to all 25 pages before discovering the Vite compilation issue. After switching to canonicalPath-based resolution, those additions were redundant and removed.
- **Fix:** Python script stripped `sourcePageUrl={import.meta.url}` from all 25 pages. Pages ended up byte-identical to their pre-task state.
- **Files modified:** All 25 src/pages/*.astro files (net zero change — add then remove)
- **Commit:** Cleanup happened before staging; pages never committed with the prop
