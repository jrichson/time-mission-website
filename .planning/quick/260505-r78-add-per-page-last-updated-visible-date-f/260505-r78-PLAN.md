Create `src/lib/last-updated.ts` exporting:

```ts
export interface LastUpdatedInfo {
    iso: string;          // YYYY-MM-DD (what goes in <time datetime="..."> and <meta>)
    human: string;        // "Last updated: May 2026"
    source: 'override' | 'git' | 'build';
}

export interface ResolveLastUpdatedInput {
    /** Optional explicit override from definePage(). MUST be 'YYYY-MM-DD'. */
    override?: string;
    /** Absolute path or file:// URL to the page's source .astro file (typically import.meta.url). */
    sourceFile?: string;
}

export function resolveLastUpdated(input: ResolveLastUpdatedInput): LastUpdatedInfo;
export function formatLastUpdatedHuman(iso: string): string;
```

Implementation rules:

1. **Override path** — if `input.override` matches `/^\d{4}-\d{2}-\d{2}$/`, return `{ iso: override, human: formatLastUpdatedHuman(override), source: 'override' }`. If override is set but malformed, throw `Error('resolveLastUpdated: override must be YYYY-MM-DD, got <value>')` (fail loud — same posture as `definePage()`'s canonicalPath guard).
2. **Git mtime path** — only if no override. Resolve `sourceFile` to a filesystem path:
   - Convert `file://...` URLs via `fileURLToPath` from `node:url`.
   - Use `node:child_process` `execFileSync('git', ['log', '-1', '--format=%cs', '--', filePath], { stdio: ['ignore', 'pipe', 'ignore'], encoding: 'utf8' })` (the `%cs` format already returns committer date as `YYYY-MM-DD`, no parsing needed).
   - Trim output. If non-empty and matches `/^\d{4}-\d{2}-\d{2}$/`, return `{ iso, human: ..., source: 'git' }`.
   - Wrap in `try/catch` — any failure (git not found, file not tracked, dirty file with no commits, non-git environment) silently falls through to build path.
3. **Build path** — final fallback. Use `new Date()` formatted to `YYYY-MM-DD` (UTC, via `toISOString().slice(0, 10)` so timezone of the build host doesn't matter).
4. `formatLastUpdatedHuman(iso)`: parse `YYYY-MM-DD`, return `"Last updated: <Month> <YYYY>"` using `Intl.DateTimeFormat('en-US', { month: 'long', year: 'numeric', timeZone: 'UTC' }).format(new Date(iso + 'T00:00:00Z'))`. The `timeZone: 'UTC'` + explicit `T00:00:00Z` prevents off-by-one in negative-UTC build environments.

Then extend `src/lib/define-page.ts` to add `lastUpdated?: string` (YYYY-MM-DD) to `SitePageMeta`. Do NOT validate the format here — `resolveLastUpdated()` owns validation so the error surfaces at the same spot as the actual usage. Keep the existing `canonicalPath` guard untouched.

Caching: not needed — Astro's static build runs each page once; git lookups per page (~28 pages) cost trivial milliseconds. Avoid premature optimization.

Constraints (per CLAUDE.md): scope commits strictly to these two files. Do not touch unrelated dirty/untracked files.
